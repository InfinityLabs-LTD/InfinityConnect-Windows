# Установщик Infinity Connect

Кастомный NSIS-установщик в фирменном тёмно-фиолетовом оформлении приложения.

## Оформление

- Графика в палитре приложения (`src/theme/colors.ts`): тёмный космос `#0B0716`,
  акценты индиго→фиолет→маджента, сетка как у `MeshBackground`, логотип «I».
- Генерируется скриптом [`scripts/gen-installer-art.py`](scripts/gen-installer-art.py)
  (Pillow) в `src-tauri/installer/`:
  - `sidebar.bmp` 164×314 — боковая панель приветствия/финала;
  - `header.bmp` 150×57 — баннер в шапке внутренних страниц.
  - Формат: BMP3, 24-бит (требование NSIS). Файлы закоммичены в репозиторий.
- Язык установки: русский (по умолчанию) + английский. Иконка — `icons/icon.ico`.
- Настройки в `src-tauri/tauri.conf.json` → `bundle.windows.nsis`.

Перегенерировать графику: `python scripts/gen-installer-art.py`.

## Подпись (издатель «Infinity Labs»)

Установщик и exe подписываются самоподписанным Authenticode-сертификатом
`CN=Infinity Labs` через [`scripts/sign.ps1`](scripts/sign.ps1) (вызывается
бандлером Tauri, `bundle.windows.signCommand`).

Сертификат хранится в секретах репозитория (base64 pfx + пароль):
- `WINDOWS_CERT_PFX_BASE64`
- `WINDOWS_CERT_PASSWORD`

Если секреты не заданы (например, сторонний контрибьютор собирает локально) —
подпись пропускается, сборка не падает.

### Важно про SmartScreen

Сертификат **самоподписанный**. Он задаёт имя издателя **Infinity Labs**
(вместо «Неизвестный издатель»), но НЕ создаёт доверенную цепочку сертификации,
поэтому фильтр Windows SmartScreen всё равно будет предупреждать при первом
запуске у пользователей.

**Полностью убрать предупреждение SmartScreen можно только реальным
code-signing сертификатом от центра сертификации:**
- **OV** (~$200–400/год) — предупреждение уходит по мере накопления репутации;
- **EV** (~$300–600/год) — доверие с первого запуска.

Когда появится реальный сертификат: заменить содержимое секретов
`WINDOWS_CERT_PFX_BASE64` / `WINDOWS_CERT_PASSWORD` на новый pfx — остальной
pipeline менять не нужно.

## Локальная проверка

```powershell
$env:WINDOWS_CERT_PFX_BASE64 = "<base64 pfx>"
$env:WINDOWS_CERT_PASSWORD   = "<пароль>"
npm run tauri build -- --bundles nsis
```

Готовый установщик: `src-tauri/target/release/bundle/nsis/*-setup.exe`.
Проверить подпись: `signtool verify /pa /v <файл>` (для самоподписанного
ожидаема ошибка доверия цепочки — это нормально, имя издателя при этом корректно).
